1) Подозрительный файл levlel13
2) Откроем его, получили набор каких-то символов - двоичный файл, ну что ж выполним его
3) ./level13
UID 2013 started us but we we expect 4242 - нам говорят, что файл выполнится только под пользавтелем с UID 4242
4) Хм, может создать пользователя с таким UID? Все попытки создать пользователя с заданными UID с помощью команд useradd и adduser потерпели неудачу.
5) Посмотрим с помощью команды  ltrace ./level13:
__libc_start_main(0x804858c, 1, 0xbffff7f4, 0x80485f0, 0x8048660 <unfinished ...>
getuid() = 2013
getuid() = 2013
printf("UID %d started us but we we expe"..., 2013UID 2013 started us but we we expect 4242) = 42
exit(1 <unfinished ...>
+++ exited (status 1) +++
Да, видно, что программы вызывает getuid(), если результат не равен 4242, то завершает работу
6) Значит вспомним корвар и будем дизассемблировать с помощью gdb
gdb ./level13
7)  disass main - дизассемблируем всю нашу историю, видно, что в строке с адресом 0x08048595 есть функция call -> она вызывает нашу функцию getuid()
В следующей строке она сравнивает ее с помощью функции cmp переменную eax. То есть по сути нам нужно подменить значение на 4242
8 ) Берем адрес строки 0x0804859a, где сравниваются значения, и ставим breakpoint:
break *0x0804859a
9) Потом запускаем с помощью команды run
10) Посмотрим какое значение там лежит print $eax
 $1 = 2013, а должно быть 4242
11) Заменим на 4242:
set $eax=4242
Продолжим с помощью команды «c» (либо с помощью next)
12) И получим токен:
Continuing.
your token is 2A31L79asukciNyi8uppkEuSx
